Three training configurations were evaluated in order to analyze the impact of
dataset size, input resolution, and data augmentation on the performance of the
QR and barcode detection model.

The first configuration represents an initial baseline model trained on a smaller
dataset. The second configuration increases the dataset size and input resolution
while still relying only on the online augmentation provided by the YOLOX
training pipeline.

augmentation generated by the augmentation pipeline described in

the model’s generalization performance.

\paragraph{Augmented  Model (v2)}

Figure~\ref{fig:augmented_training} illustrates the training behaviour of the
augmented model (v2), which was trained using additional offline data
augmentation in combination with the online augmentations applied during
training.

As shown in Figure~\ref{fig:augmented_training}a, the training loss decreases
rapidly during the initial epochs and stabilizes as training progresses. The
individual loss components follow a similar decreasing trend, indicating stable
optimization and successful convergence of the model.

The validation mAP curve in Figure~\ref{fig:augmented_training}b demonstrates a
steady improvement throughout training, eventually stabilizing around 0.9. This
stabilization triggered the early stopping criterion.
One notable difference between the v1 and v2 configurations can be observed in
the fluctuations of the validation mAP curves. While
Figure~\ref{fig:improved_training}b appears as a relatively homogeneous band,
indicating a more uniform variance over time,
Figure~\ref{fig:augmented_training}b shows more distinct oscillations between
evaluation steps.

This suggests that the v2 model was terminated while still experiencing
noticeable variance in validation performance, likely due to the increased
variability introduced by the augmentation pipeline. In contrast, the v1 model
exhibits a more stable convergence behaviour.

Finally, the learning rate schedule shown in
Figure~\ref{fig:augmented_training}c follows the same warm-up and gradual decay
pattern as in the previous experiments.

Unlike the v1 configuration, the learning rate in the v2 configuration was
terminated earlier due to the early stopping criterion and therefore did not
reach its final stabilization phase. As illustrated in
Figure~\ref{fig:augmented_training}c, the learning rate was still gradually
decreasing at the point where training was stopped.

One possible explanation is that the additional data augmentation increased the
variability of the training data, which can slow down convergence. In the v2
configuration, both offline and online augmentation were applied, potentially
making the optimization process more complex.

The final stage of the learning rate schedule is typically important because
very small learning rates allow the optimizer to fine-tune the model
parameters. When training stops before this phase is reached, the model may not
fully benefit from this fine-tuning stage, which could lead to slightly
suboptimal performance.

\caption{Augmented (v2) configuration.}

\caption{Side-by-side comparison of the precision–recall curves for the improved (v1) and augmented (v2) configurations.}
\label{fig:pr_curves} 
\end{figure}

Figure~\ref{fig:pr_curves} shows the precision–recall curves for the improved
(v1) and augmented (v2) configurations. Both models achieve high precision
across most recall levels, indicating strong detection performance.

However, the v1 configuration achieves a slightly higher average precision ($AP \approx 0.98$) compared to the augmented v2 configuration ($AP \approx 0.96$).
While both curves remain close to the top-right corner of the plot, the v2
model exhibits a more noticeable drop in precision at very high recall values.

The area under the precision-recall curve, also called PR-AUC, is given by the formula:

\[
AP = \sum_{n} (R_n - R_{n-1}) P_n
\]

Where:
\begin{itemize}
    \item $P_n$ is the precision at the $n$-th threshold.
    \item $R_n$ is the recall at the $n$-th threshold.
    \item $(R_n - R_{n-1})$ represents the change in recall, which corresponds to the width of each "step" observed in the PR curves (e.g., Figure~\ref{fig:pr_curves}).
\end{itemize}

Precision and Recall are defined based on True Positives (TP), False Positives (FP), and False Negatives (FN):

\begin{align*}
Precision &= \frac{TP}{TP + FP} & Recall &= \frac{TP}{TP + FN}
\end{align*}

By analyzing Figure~\ref{fig:pr_curves}b and considering the mathematical definitions of precision and recall, the behaviour suggests that the augmented model tends to miss more detections at certain confidence thresholds. From the mathematical definition of recall, an increase in false negatives leads to a lower recall value. As recall decreases, the area under the precision–recall curve is reduced, resulting in a lower average precision (AP). 

This observation indicates that the augmented configuration exhibits a more
conservative prediction behaviour, where the model becomes more cautious and
misses some detections. In contrast, the v1 configuration maintains a more
stable precision–recall trade-off, resulting in a higher overall average
precision.


To obtain a more detailed analysis of the precision–recall behaviour, confusion
matrices were generated for both models. Both models were tested on the same
unseen test set, consisting of 407 real-life images, in order to ensure a fair
comparison.

\begin{figure}[H]
\centering
\includegraphics[width=12cm]{figures/images/chapter-13/qr-bar/baseline-model-v2/confusion_matrix_0.3.png}
\caption{Confusion matrix for the improved (v1) configuration.}
\label{fig:confusion_matrix_v1}
\end{figure}

Figure~\ref{fig:confusion_matrix_v1} illustrates the confusion matrix for the
v1 configuration and demonstrates strong detection performance.
The model correctly detected 970 barcodes while missing 54 barcodes (false
negatives) and producing 20 false positive detections.

\begin{align*}
Precision_{v1} &= \frac{TP}{TP + FP} = \frac{970}{970 + 20} \approx 0.9798,
&
Recall_{v1} &= \frac{TP}{TP + FN} = \frac{970}{970 + 54} \approx 0.9473
\end{align*}

These values indicate that the v1 configuration achieves very high precision,
meaning that most detected barcodes are correct. At the same time, the recall
remains high, showing that the model successfully detects the majority of
barcodes present in the images.

\begin{figure}[H]
\centering
\includegraphics[width=12cm]{figures/images/chapter-13/qr-bar/augmented-model-v2/confusion_matrix_0.3.png}
\caption{Confusion matrix for the augmented (v2) configuration.}
\label{fig:confusion_matrix_v2}
\end{figure}

The v2 configuration also achieves a high number of true positive detections,
but performs slightly worse than the v1 configuration. In total, 155 barcodes
were missed (false negatives), which supports the observations from the
precision–recall analysis. The augmented model therefore behaves more
conservatively, resulting in a larger number of missed detections.

This difference is reflected in the recall values, where the v2 configuration
achieves a noticeably lower recall than the v1 configuration.

\begin{align*}
Precision_{v2} &= \frac{TP}{TP + FP} = \frac{959}{959+31} \approx 0.9687,
&
Recall_{v2} &= \frac{TP}{TP + FN} = \frac{959}{959+155} \approx 0.8609
\end{align*}

Although the precision remains relatively high, the recall is noticeably lower
compared to the v1 configuration. This indicates that the augmented model misses
more barcodes during detection, which aligns with the observations from the
precision–recall analysis and the confusion matrix.

Overall, the confusion matrix analysis confirms the results observed in the
precision–recall curves: the improved v1 configuration provides a better
balance between precision and recall, while the augmented v2 configuration
tends to miss more detections.

Table~\ref{tab:model_comparison_threshold} summarizes the detection performance of both models at a practical
confidence threshold of 0.3. At this operating point, the baseline
configuration achieves higher precision, recall, and F1-score compared to the
augmented configuration.

The augmented model produces a noticeably higher number of false detections
and missed objects, which results in a lower overall F1-score. These results
are consistent with the confusion matrix and precision–recall analyses
presented earlier, where the augmented configuration showed a more
conservative detection behaviour and missed more barcodes.

The Average Precision comparison in Table~\ref{tab:ap_comparison} shows that
the v1 configuration achieves slightly higher AP values across the evaluated
thresholds compared to the augmented model.
This indicates that the augmented configuration may generalize slightly better
when evaluated across the full range of confidence thresholds.

Overall, while augmentation slightly improves the global AP metric, the
improved v1 configuration provides better practical detection performance at
the selected operating threshold. The confusion matrix and recall values
indicate that the augmented model misses significantly more barcodes, suggesting
a more conservative prediction behaviour.

One possible explanation is that the combination of offline and online
augmentation introduced excessive variability into the training data, which may
have made the optimization process more difficult for the model. As a result,
the model becomes more cautious and fails to detect a larger number of objects.

Therefore, the v1 configuration represents the most reliable model for
deployment in the proposed system.
